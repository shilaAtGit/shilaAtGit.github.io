学习theme-mainroad的记录

Shila Jin
2024-02-28
网络技术
记录一下学习theme的过程，先学习的是自己使用的mainroad。 mainroad是hugo的众多theme中的一个，比较适合简单使用，写自己的blog。 首先，先贴一下hugo的主页： hugo主页 从hugo主页的菜单themes主页进入，可以选择自己喜欢的theme，每个theme都有详细的自己的下载、使用教程。使用教程的主要内容为config和front matter。 其中，对于config的配置，新版本的hugo已经放到了hugo.toml中，不在放在config下，也可以自己新建config目录和config.toml文件存放，但是没啥必要。 theme主要的定制能力，都体现在front matter上。front matter的定义，是一个theme的样式。比如在我的所有md文档中，都要复制mainroad提供的frontmatter到文件头。 front matter的内容提供了定制，可以自己根据需求进行调整。比如页面包涵那些模块，模块的摆放位置等。

申请github账户

Shila Jin
2024-02-19
网络技术
介绍一下如何实现github的账户申请，以及在github上建立第一个仓库。 这个仓库后续要用来发布个人主页，个人主页的搭建可以参考我另外一篇文章。学习使用hugo笔记。 开篇语 github是不太稳定的，有的时候无法访问，如果个人愿意出钱，可以选择其他的云服务，比如阿里云、腾讯云、华为云等，这些付费的专业云可以获得比较好的用户体验。 本文主导的是免费的策略，顺带熟悉github的发布。 github之前被墙过，但是现在基本是允许访问的，一些教你翻墙、hosts设置代理等访问git的方式，可以先放弃了。如果后期github再次被墙了，需要寻找相应的访问办法。 申请github账号： 如果有账号直接登录就好了。 gitlab的平台界面和操作路径会更新，所以有的时候需要自己找一下操作位置。 访问gitlab，点击右上角的sign up。 然后填写注册信息就好了，注册信息很简单，包括email，用户名，口令。后续页面会收集其他信息，以及推荐选择付费版本，记住选择免费即可。 第一步：新建github仓库 1、登陆在gitlab，点击左侧的new按键，新建一个repository。 2、填写新的仓库的信息，注意不要修改任何信息，需要填写的只有Repository Name. Name不能乱写，必须是.github.io，比如我的github的名字叫zhangsan123，Repository Name就是zhangsan123.github.io。 另外，不要因为前面就是owner，就不写owner了 就此，github仓库就创建好了，后面只需要将本地代码提交到这个仓库就可以了。

配置本地的github的SSH

Shila Jin
2024-02-19
网络技术
简单讲解一下如何生成本地的ssh key，并配置到github账户，以实现本地代码推送到远端。 不配置这一步，是无法通过命令行推送代码的。 配置本地git 配置本地的git环境，才能将本地代码提交到github，这里讲如何通过命令行进行提交的准备工作。 一、生成ssh key git提交，需要与远程仓库通过ssh安全通道进行提交，所以要先生成本地的ssh key，并且配置到服务器，才可以实现提交。 1、生成本地ssh key。 这里“你的邮箱”，指的是你注册github时候使用的邮箱 ssh-keygen -t rsa -C “你的邮箱” 生成过程中，会提示输入密码等一系列操作，全部用回车即可。 2、执行 ssh -v git@github.com 3、执行 ssh-agent -s 4、执行 ssh-add ~/.ssh/id_rsa 这个时候出现的Identity added…这里的路径就是证书文件路径 5、打开证书文件路径，找到id_rsa.pub文件，用文本阅读器打开，看到ssh-rsa开头的一段base64公钥，全文复制 。 二、配置ssh key 1、浏览器登录到你的github账号，点击右侧头像，选择settings 2、在settings中选择SSH and GPG keys 3、选择New SSH key 4、把刚才复制的公钥复制到key的文本框中，title可以随便起一个，完成后点击Add SSH key 至此，就完成了ssh key的配置，可以验证一下，在命令行输入： ssh -T git@github.com
