使用git搭建免费个人主页

Shila Jin
2024-02-19
hugo, 网络技术
作为一个程序员，当然要有自己的主页，使用hugo是非常方便的方式. 具体如何进行hugo项目搭建，参考我另外一篇文章。学习使用hugo笔记 完成了个人主页的搭建，就需要有一个互联网地址，能够实现public的访问，比如允许朋友查看，分享什么。本文介绍如何使用github来搭建一个免费的个人主页。 开篇语 github是不太稳定的，有的时候无法访问，如果个人愿意出钱，可以选择其他的云服务，比如阿里云、腾讯云、华为云等，这些付费的专业云可以获得比较好的用户体验。 本文主导的是免费的策略，顺带熟悉github的发布。 github之前被墙过，但是现在基本是允许访问的，一些教你翻墙、hosts设置代理等访问git的方式，可以先放弃了。如果后期github再次被墙了，需要寻找相应的访问办法。 申请github账号： 参考申请github账户 第二步：配置本地git 参考配置本地的github的SSH 第三步：推送本地代码到github 1、要根据发布地址，生成hugo最终发布的代码。使用hugo命令可以生成发布代码，并放入项目下的public目录。这里要指定发布地址，否则后续修改比较麻烦。 在项目根目录下执行命令： hugo –baseURL=https://shilaAtGit.github.io.git URL=后面的内容，就是之前在git账户申请过程中，新建的仓库的https地址，根据之前新建的规则，地址为：https://<你的github账号>.github.io 命令执行后，会在项目根目录下生成一个public目录 1.1、这里要格外注意： 把自己不想发布的文件夹先删除再执行上面的生成过程（生成并推送之后可以再从垃圾站里找回来） 2、进入public目录：cd public 3、逐一执行以下命令： git init //每次修改了代码重新生成public后，都要从这里开始执行 git remote add origin git@github.com:shilaAtGit/shilaAtGit.github.io.git git add . git commit -m “提交说明” git push origin main //也许需要加 -f，因为是个人开发，所以直接强制把本地的代码更新到远端。 很多教程在这里push的是master，也会教你如何从main改为master，但是去掉master是官方行为，用main挺好不影响使用。 4、这样就完成了push到远端github，需要稍微等一会才能访问。网络墙也许会给你带来困惑，比如我的网络可以访问github的管理页面，但是无法访问我自己的主页，就很奇怪。 提供一些参考的git语句： git pull origin main git branch -l git remote add origin git@github.com:shilaAtGit/shilaAtGit.github.io.git 第四步：发布和重新发布 很多时候会发现自己的发布并没有成功，那就需要重新发布，在github管理界面操作即可。 1、登陆github账户，进入项目 2、选择actions 3、选择一个workflow进入 4、点击reRun 5、观察如下，就是发布成功了 终章，搞定啦 至此，一个免费的基于github服务的个人主页就完成了，剩下的就是根据hugo来进行MD格式内容的编写了。 每次编写后，使用hugo server命令来在本地查看效果，使用hugo public来打包，使用git 推送到远端。

学习使用hugo笔记

Shila Jin
2024-02-15
hugo, 程序员
写一篇笔记，记录hugo的学习过程，以便遗忘时复阅。注：文中资源、命令有时限性，需要依时更新之。 开篇语 hugo是基于go语言的快速搭建网站的框架，然而实现目标——搭建网站——却不需要有任何go语言的学习过程。 需要的能力是：下载能力，命令行输入命令能力，阅读Doc能力，这些基础能力具备后，只要会写MD格式的网站内容文件就可以了。 因此，hugo是搭建个人网站最优的方案，因为即使是code大神——为了记录信息才是核心目标，也完全没必要自己去费时建设。 hugo主页：hugo 准备工作： 安装git（本来就有） 安装brew（很久以前的工具，无法升级，为了修复它话费了大量的时间） 安装hugo（有了brew一切都很容易） 第一步：修复homebrew 1、输入brew update报错，尝试各种百度方法，最后卡在提示ruby版本与操作系统不符合，表现为更新ruby失败，手动更新后提示操作系统版本旧。因为我无法升级操作系统，所以困住于此。 2、使用如下方法，修复 命令： /bin/zsh -c “$(-fsSL https://gitee.com/cunkai/HomebrewCN/raw/master/Homebrew.sh)” 命令会引导命令终端进入交互模式，都选1-中科大镜像，即可。 完成brew的完整安装后，输入brew update进行升级。 第二步：安装go go有自己的可视化安装包，下载并安装即可。 第三步：安装hugo 命令：brew install hugo 参考安装hugo 第四步：生成项目并启动服务 hugo new site quickstart //建立一个叫做quickstart的项目，可以用任意名称 cd quickstart //进入项目根目录 git init //初始化git git submodule add https://github.com/theNewDynamic/gohugo-theme-ananke.git themes/ananke //下载ananke theme echo “theme = ‘ananke’” » hugo.toml //启用theme hugo server //启动服务，日志输出中会显示服务地址http://localhost:1313 操作非常简单，至此就完成了整个网站的建设。具体可以参考官网搭建hogo项目 第五步：更换喜欢的theme hugo提供很多的theme，可以根据目标选择。参考themes 1、下载theme，并放到项目的themes目录下，参考语句：git clone https://github.com/vimux/mainroad.git themes/mainroad 2、更新hugo.toml，在项目根目录找到 theme = ‘ananke’，修改为theme = ‘mainroad’
