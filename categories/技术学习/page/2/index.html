配置本地的github的SSH

Shila Jin
2024-02-19
技术学习
简单讲解一下如何生成本地的ssh key，并配置到github账户，以实现本地代码推送到远端。 不配置这一步，是无法通过命令行推送代码的。 配置本地git 配置本地的git环境，才能将本地代码提交到github，这里讲如何通过命令行进行提交的准备工作。 一、生成ssh key git提交，需要与远程仓库通过ssh安全通道进行提交，所以要先生成本地的ssh key，并且配置到服务器，才可以实现提交。 1、生成本地ssh key。 这里“你的邮箱”，指的是你注册github时候使用的邮箱 ssh-keygen -t rsa -C “你的邮箱” 生成过程中，会提示输入密码等一系列操作，全部用回车即可。 2、执行 ssh -v git@github.com 3、执行 ssh-agent -s 4、执行 ssh-add ~/.ssh/id_rsa 这个时候出现的Identity added…这里的路径就是证书文件路径 5、打开证书文件路径，找到id_rsa.pub文件，用文本阅读器打开，看到ssh-rsa开头的一段base64公钥，全文复制 。 二、配置ssh key 1、浏览器登录到你的github账号，点击右侧头像，选择settings 2、在settings中选择SSH and GPG keys 3、选择New SSH key 4、把刚才复制的公钥复制到key的文本框中，title可以随便起一个，完成后点击Add SSH key 至此，就完成了ssh key的配置，可以验证一下，在命令行输入： ssh -T git@github.com

如何设置github代理地址

Shila Jin
2018-08-21
技术学习
作为国人在研发中一个痛点就是无法自由使用很多全球资源，一些公司提供相应的解决方案，但是一些公司因为性质问题不能提供，只能由程序员自行解决。 如何访问guihub就是其中一个。方法目前比较靠谱的就是如下，以本人mac系统为例进行阐述： 1、首先，即使使用代理，github也是不太稳定的，这个只能忍了。 2、可用的github代理经常会发生变化，所以及时更新很重要。 3、访问https://raw.hellogithub.com/hosts，用浏览器打开此地址即可下载文件，用本地文本工具打开，即可看到最新的github相关的代理地址。 4、更新本地的hosts文件，mac的文件位置在：/etc/hosts. 5、修改hosts文件需要管理员口令，有的工具不支持输入管理员口令进行修改，使用sublimeText工具可以编辑。 6、使hosts的修改生效：sudo killall -HUP mDNSResponder

CFX生成WS客户端

Shila Jin
2014-02-01
技术学习
记录一下如何根据CFX生成web servers的client端代码 WSDL介绍 WSDL 可描述网络服务（Web Services） WSDL 指网络服务描述语言 (Web Services Description Language)。 WSDL 是一种使用 XML 编写的文档。这种文档可描述某个 Web service。它可规定服务的位置，以及此服务提供的操作（或方法）。 下载CFX 下载地址：http://www.apache.org/dyn/closer.cgi?path=/cxf/2.7.11/apache-cxf-2.7.11.zip 将CXF压缩包解压到本地目录 配置环境变量 要根据操作系统的不同来配置。 配置环境变量 在CLASSPATH %CXF_HOME%\lib 在PATH中添加 %CXF_HOME%\bin 验证环境变量 验证：命令行输入： wsdl2java -v 结果显示：wsdl2java -Apache CXF 2.7.11说明配置成功 wsdl2java工具使用 使用命令行输入： wsdl2java -frontend jaxws21 -p com.test.client -d D:\ -client -autoNameResolution http://192.168.0.118:8080/framework/ws/helloService?wsdl 说明: -p 包名 -d 文件生成的本地目录 http://192.168.0.118:8080/framework/ws/helloService?wsdl 发布的服务访问路径 另外，记录： 以下的方法我没有用过，记录参考，万一用到 使用本地wsdl文件生成代码 使用命令行输入：wsdl2java -p com.test.client -d D: -verbose “D:\helloService.xml” 说明：-p 包名 -d生成文件路径 -verbose “wsdl文件路径”
